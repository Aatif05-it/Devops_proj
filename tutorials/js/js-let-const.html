JS Let & Const

let and const create blockâ€‘scoped variables, fixing many issues caused by the older var keyword.

When to use each

Use const for values that should not be reassigned and let for values that will change over time. 

<!DOCTYPE html>
<html>
<body>

  <pre id="info"></pre>

  <script>
    const pi = 3.14;      // constant value
    let counter = 0;      // will change

    counter = counter + 1;

    {
      let counter = 100;  // different variable (block scope)
      // this counter exists only inside this block
    }

    const text =
      "pi (const): " + pi + "\\n" +
      "counter (let): " + counter;

    document.getElementById("info").textContent = text;
  </script>

</body>
</html>

Both let and const are hoisted but not initialized, so using them before declaration causes a ReferenceError instead of silent bugs.
