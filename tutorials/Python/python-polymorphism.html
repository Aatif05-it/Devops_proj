Python Polymorphism

Polymorphism means â€œmany formsâ€ and allows the same interface to be used for different underlying types.

In Python this is often done via method overriding and duck typing.

Method overriding in inheritance

class Animal:
    def speak(self):
        print("Some generic sound")

class Dog(Animal):
    def speak(self):
        print("Woof!")

class Cat(Animal):
    def speak(self):
        print("Meow!")

animals = [Dog(), Cat(), Animal()]
for a in animals:
    a.speak()

Each subclass provides its own version of speak(), but the same call a.speak() works for every object.

Duck typing

class PdfReport:
    def render(self):
        print("Rendering PDF...")

class HtmlReport:
    def render(self):
        print("Rendering HTML...")

def generate(report):
    # works with any object that has a render() method
    report.render()

generate(PdfReport())
generate(HtmlReport())

Python focuses on behavior instead of types: â€œif it walks like a duck and quacks like a duck, treat it like a duckâ€.
