Python Context Managers

Context managers handle setup and cleanup around a block of code, most commonly used with the with statement.

They ensure resources like files, locks, or network connections are always released properly.

Using with for files

with open("data.txt", "r", encoding="utf-8") as f:
    for line in f:
        print(line.strip())

When the with block ends, the file is automatically closed, even if an exception occurs.

Custom class-based context manager

class Timer:
    def __enter__(self):
        import time
        self.start = time.perf_counter()
        return self

    def __exit__(self, exc_type, exc_val, exc_tb):
        import time
        self.end = time.perf_counter()
        self.elapsed = self.end - self.start
        print(f"Elapsed: {self.elapsed:.4f} seconds")

with Timer():
    total = sum(range(1_000_000))

A context manager class implements __enter__ and __exit__, which run at the start and end of the with block.
