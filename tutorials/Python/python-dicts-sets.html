Python Dictionaries & Sets

Dictionaries store data as key–value pairs, while sets store unique, unordered values.

Dictionaries are ideal for fast lookups by key; sets are great for membership tests and removing duplicates.

Creating and using dictionaries

# Dictionary literal
user = {
    "name": "Alice",
    "age": 21,
    "skills": ["Python", "SQL"]
}

print(user["name"])          # Alice
user["age"] = 22             # update value
user["city"] = "Mumbai"      # add new key
print(user.get("email"))     # None (safe access)

Use square brackets to read or update a key, and dict.get() when you want a safe lookup that returns None or a default instead of raising an error.

Dictionary methods

user = {"name": "Alice", "age": 21}

print(user.keys())       # dict_keys(['name', 'age'])
print(user.values())     # dict_values(['Alice', 21])
print(user.items())      # dict_items([('name', 'Alice'), ('age', 21)])

for key, value in user.items():
    print(key, "→", value)

removed_age = user.pop("age")   # remove key and return value
user.clear()                    # remove all items

keys(), values(), and items() let you iterate over a dictionaryâ€™s structure, while pop() and clear() remove entries.

Creating and using sets

# Set literal (duplicates removed)
langs = {"python", "java", "python", "c++"}

print(langs)                 # {'python', 'java', 'c++'}

langs.add("go")
langs.remove("java")         # raises KeyError if not present
langs.discard("rust")        # safe: no error if missing

print("python" in langs)     # membership test

Sets automatically drop duplicates and support fast membership checks using in and not in.

Set operations

backend = {"python", "java", "go"}
frontend = {"javascript", "typescript", "python"}

print(backend | frontend)   # union
print(backend & frontend)   # intersection
print(backend - frontend)   # difference
print(backend ^ frontend)   # symmetric difference

Use union, intersection, and difference to combine or compare sets of tags, permissions, or IDs.
