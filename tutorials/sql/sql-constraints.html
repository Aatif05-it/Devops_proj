SQL Constraints Basics

Constraints enforce rules on table columns so that data stays valid and consistent, such as requiring unique ids or preventing missing values.

Common constraint types are PRIMARY KEY, FOREIGN KEY, UNIQUE, CHECK, DEFAULT, and NOT NULL.

Declaring constraints

Constraints can be defined inline with the column or as separate tableâ€‘level definitions. 

CREATE TABLE Customers (
  CustomerID   INT PRIMARY KEY,
  CustomerName VARCHAR(100) NOT NULL,
  Email        VARCHAR(100) UNIQUE,
  Age          INT CHECK (Age >= 18)
);

Here the primary key uniquely identifies each row, the name cannot be NULL, the email must be unique, and the age must be at least 18.

Adding constraints later

Existing tables can gain new constraints using ALTER TABLE, which is common as schemas evolve. 

ALTER TABLE Customers
ADD CONSTRAINT chk_age CHECK (Age >= 18);

Giving constraints descriptive names makes it easier to reference or drop them in future migrations.
