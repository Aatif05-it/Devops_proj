SQL WHERE & Operators

The WHERE clause filters rows so that only records matching given conditions are returned or affected.

Comparison and special operators such as =, >, <, BETWEEN, LIKE, and IN express these conditions.

WHERE with comparison operators

This example selects customers from France whose customer id is greater than 100. 

SELECT CustomerID, CustomerName, Country
FROM Customers
WHERE Country = 'France'
  AND CustomerID > 100;

Operators like > and >= are often combined with equality tests to narrow results to precise ranges.

BETWEEN, LIKE, and IN

These operators help express ranges, pattern matches, and membership tests more clearly than multiple comparisons. 

-- Between a numeric range
SELECT *
FROM Orders
WHERE TotalAmount BETWEEN 100 AND 500;

-- Pattern match for names starting with 'A'
SELECT *
FROM Customers
WHERE CustomerName LIKE 'A%';

-- Match any of several countries
SELECT *
FROM Customers
WHERE Country IN ('France', 'Germany', 'Spain');

Using BETWEEN, LIKE, and IN keeps queries shorter and easier to maintain than writing many separate conditions.
