SQL Syntax & Data Types

SQL statements are written as keywords followed by identifiers such as table and column names, ending with a semicolon in many database systems.

Each column in a table has a data type that controls what kind of values it can store, such as integers, decimals, dates, or text strings.

Basic syntax example

This example shows a simple statement that selects two columns from a table, following the pattern KEYWORDS → identifiers → semicolon. 

SELECT CustomerName, City
FROM Customers;

SQL is generally case-insensitive for keywords, so select and SELECT behave the same, but using uppercase keywords improves readability.

Common data types

Databases support many data types, but most queries rely on numeric, character, and date/time types as a foundation. 

-- Example column definitions
INT              -- whole numbers
DECIMAL(10, 2)   -- fixed decimal money values
VARCHAR(100)     -- variable-length text up to 100 characters
DATE             -- calendar date (YYYY-MM-DD)
TIMESTAMP        -- date and time
BOOLEAN          -- true / false (some systems use TINYINT)

Choosing the smallest data type that fits your data helps save storage and can significantly improve query performance on large tables.
