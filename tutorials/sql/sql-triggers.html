SQL Triggers

Triggers automatically execute SQL code in response to INSERT, UPDATE, or DELETE events on a table, enforcing rules without application code.

They maintain audit trails, update related tables, and validate data changes at the database level.

AFTER INSERT trigger

This trigger logs every new order to an audit table with timestamp and user info. [web:477][web:478] 

CREATE TRIGGER OrderAudit
AFTER INSERT ON Orders
FOR EACH ROW
BEGIN
  INSERT INTO OrderAuditLog (OrderID, Action, UserID, Timestamp)
  VALUES (NEW.OrderID, 'INSERT', USER(), NOW());
END;

NEW refers to the inserted row; the trigger fires automatically after each insert.

BEFORE UPDATE trigger

This prevents price decreases below cost and logs attempted changes. 

CREATE TRIGGER PreventPriceDrop
BEFORE UPDATE ON Products
FOR EACH ROW
BEGIN
  IF NEW.Price < NEW.Cost THEN
    SIGNAL SQLSTATE '45000' 
    SET MESSAGE_TEXT = 'Price cannot be less than cost';
  END IF;
END;

BEFORE triggers can modify NEW values or block the operation entirely with SIGNAL.
