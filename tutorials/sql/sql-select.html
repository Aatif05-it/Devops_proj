SQL SELECT

The SELECT statement retrieves data from one or more tables and returns a result set of rows that match your query.

A basic query has the form SELECT columns FROM table, and additional clauses like WHERE and ORDER BY refine which rows are returned and how they are sorted.

Select specific columns

This example returns only the customer name and city from the Customers table. 

SELECT CustomerName,
       City
FROM Customers;

Selecting only the needed columns keeps queries faster and outputs smaller, which matters when tables contain many fields or millions of rows.

Select all columns

Using * is convenient while exploring data, but is less ideal in production code because it may return unnecessary columns. 

SELECT *
FROM Customers;

As schemas evolve, SELECT * can accidentally pull new columns that a client application does not expect, so most style guides recommend explicit column lists.
