SQL Subqueries Basics

A subquery is a query nested inside another statement, often in the WHERE, FROM, or SELECT clause.

Subqueries can return a single value, a single row, or a full result set that other parts of the query use as input.

Subquery in WHERE

This example finds customers whose total spending is greater than a fixed threshold by using a subquery that sums orders. 

SELECT CustomerID, CustomerName
FROM Customers
WHERE CustomerID IN (
  SELECT CustomerID
  FROM Orders
  GROUP BY CustomerID
  HAVING SUM(TotalAmount) > 1000
);

The inner query computes total spending per customer, and the outer query restricts the customers table to those ids.

Subquery in FROM (derived table)

Placing a subquery in the FROM clause creates a derived table that can be further filtered or joined. 

SELECT
  t.CustomerID,
  t.TotalSpent
FROM (
  SELECT CustomerID, SUM(TotalAmount) AS TotalSpent
  FROM Orders
  GROUP BY CustomerID
) AS t
WHERE t.TotalSpent > 1000;

This pattern is helpful when you want to reuse an aggregated result or apply additional filters after computing totals.
