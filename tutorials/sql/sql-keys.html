SQL Primary & Foreign Keys

A primary key uniquely identifies each row in a table, while a foreign key references the primary key of another table to link related data.

Good key design enables reliable joins and prevents orphaned rows by enforcing referential integrity.

Defining a primary key

A primary key can be declared inline with a column or as a separate tableâ€‘level constraint. 

CREATE TABLE Customers (
  CustomerID   INT PRIMARY KEY,
  CustomerName VARCHAR(100) NOT NULL
);

Primary key columns must be unique and not NULL so that every row is addressable by a single value.

Foreign key relationship

Foreign keys ensure that child rows reference existing parent rows, such as orders pointing to valid customers. 

CREATE TABLE Orders (
  OrderID    INT PRIMARY KEY,
  OrderDate  DATE NOT NULL,
  CustomerID INT NOT NULL,
  CONSTRAINT fk_orders_customers
    FOREIGN KEY (CustomerID)
    REFERENCES Customers(CustomerID)
);

With this constraint in place, inserting an order for a nonâ€‘existent customer id will fail instead of silently corrupting data.
