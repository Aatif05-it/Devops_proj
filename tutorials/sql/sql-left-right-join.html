SQL LEFT & RIGHT JOIN

A LEFT JOIN returns all rows from the left table and any matching rows from the right table, filling missing matches with NULL.

A RIGHT JOIN keeps all rows from the right table instead; many queries can use LEFT JOIN by swapping table positions.

LEFT JOIN example

This query shows every customer and their orders, including customers who have never placed an order. 

SELECT
  c.CustomerName,
  o.OrderID,
  o.OrderDate
FROM Customers AS c
LEFT JOIN Orders AS o
  ON c.CustomerID = o.CustomerID;

For customers without orders, the columns from Orders are NULL, which is helpful when searching for inactive accounts.

RIGHT JOIN example

A right join guarantees that every order appears, even if the related customer row is missing. 

SELECT
  c.CustomerName,
  o.OrderID,
  o.OrderDate
FROM Customers AS c
RIGHT JOIN Orders AS o
  ON c.CustomerID = o.CustomerID;

Some databases do not support RIGHT JOIN, so developers often rewrite such queries as left joins with tables reversed.
