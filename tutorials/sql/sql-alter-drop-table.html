SQL ALTER & DROP TABLE

The ALTER TABLE statement modifies an existing table by adding, changing, or removing columns and constraints. [web:312][web:321]

The DROP TABLE command deletes a table and all of its data, so it must be used carefully, usually only in development or after backups. [web:321][web:318]

Add and drop a column

This example adds an Email column to Customers and later removes it. [web:312][web:318] 

-- Add a new column
ALTER TABLE Customers
ADD Email VARCHAR(100);

-- Drop an existing column
ALTER TABLE Customers
DROP COLUMN Email;

Many systems support multiple column changes in one statement, but some dialects require separate ALTER TABLE commands per change.

Drop a table

This example removes the Orders table completely; using IF EXISTS avoids errors if the table is missing. 

DROP TABLE IF EXISTS Orders;

When foreign keys reference a table, you may need to drop or update those constraints before dropping the table, depending on the database.
