SQL Set Operators

Set operators extend UNION and UNION ALL with INTERSECT and EXCEPT (or MINUS), letting you compare result sets.

These operators require the participating queries to return the same number of columns with compatible data types in the same order.

INTERSECT example

This query finds cities that appear in both the Customers and Suppliers tables. 

SELECT City FROM Customers
INTERSECT
SELECT City FROM Suppliers;

Only cities present in both result sets are returned, with duplicates removed automatically.

EXCEPT / MINUS example

EXCEPT (or MINUS in some databases) returns rows from the first query that do not appear in the second. 

SELECT City FROM Customers
EXCEPT
SELECT City FROM Suppliers;
