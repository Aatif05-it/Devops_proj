SQL Basic Joins

Joins combine columns from two or more tables based on related values, such as a customer id shared between Customers and Orders.

The main join types are INNER, LEFT, RIGHT, and FULL; each controls whether unmatched rows from either table appear in the result set.

Inner join overview

An inner join returns only rows that have matching keys in both tables. 

SELECT
  c.CustomerName,
  o.OrderID,
  o.OrderDate
FROM Customers AS c
INNER JOIN Orders AS o
  ON c.CustomerID = o.CustomerID;

If a customer has no orders, that customer does not appear in the result because there is no matching row in Orders.

Join types summary

Thinking about joins in terms of Venn diagrams can help: INNER is the intersection, LEFT and RIGHT keep one full side, and FULL keeps everything. 

-- Generic syntax
SELECT ...
FROM A
JOIN B ON A.key = B.key;       -- INNER JOIN

SELECT ...
FROM A
LEFT JOIN B ON A.key = B.key;  -- LEFT OUTER JOIN

SELECT ...
FROM A
RIGHT JOIN B ON A.key = B.key; -- RIGHT OUTER JOIN

SELECT ...
FROM A
FULL JOIN B ON A.key = B.key;  -- FULL OUTER JOIN
