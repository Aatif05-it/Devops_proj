SQL CTEs (Common Table Expressions)

CTEs define temporary result sets within a single query using WITH, improving readability for complex multi-step logic.

Unlike subqueries, CTEs can be referenced multiple times and support recursion for hierarchical data.

Simple CTE example

This CTE breaks a complex query into readable steps: first find high-value customers, then their recent orders. 

WITH HighValueCustomers AS (
  SELECT CustomerID, SUM(TotalAmount) AS LifetimeValue
  FROM Orders 
  GROUP BY CustomerID 
  HAVING LifetimeValue > 5000
),
RecentOrders AS (
  SELECT o.*
  FROM Orders o
  JOIN HighValueCustomers h ON o.CustomerID = h.CustomerID
  WHERE o.OrderDate >= DATE_SUB(CURDATE(), INTERVAL 30 DAY)
)
SELECT * FROM RecentOrders ORDER BY OrderDate DESC;

Each CTE builds on the previous one, making the logic easier to follow than nested subqueries.

Recursive CTE for hierarchies

Recursive CTEs traverse tree structures like employee-manager relationships or product categories. 

WITH RECURSIVE EmployeeHierarchy AS (
  -- Anchor: Top-level employees (no manager)
  SELECT EmployeeID, EmployeeName, ManagerID, 1 AS Level
  FROM Employees 
  WHERE ManagerID IS NULL
  
  UNION ALL
  
  -- Recursive: Employees reporting to previous level
  SELECT e.EmployeeID, e.EmployeeName, e.ManagerID, eh.Level + 1
  FROM Employees e
  JOIN EmployeeHierarchy eh ON e.ManagerID = eh.EmployeeID
)
SELECT * FROM EmployeeHierarchy ORDER BY Level, EmployeeName;

The UNION ALL combines the anchor (base case) with recursive steps until no more rows match.

Multiple CTEs in one query

CTEs can run in parallel and reference each other, creating modular query pipelines. 

WITH CustomerStats AS (
  SELECT CustomerID, COUNT(*) AS OrderCount, AVG(TotalAmount) AS AvgOrder
  FROM Orders GROUP BY CustomerID
),
ProductStats AS (
  SELECT ProductID, COUNT(*) AS OrderCount
  FROM OrderItems GROUP BY ProductID
)
SELECT c.CustomerName, cs.OrderCount, cs.AvgOrder
FROM Customers c
JOIN CustomerStats cs ON c.CustomerID = cs.CustomerID
WHERE cs.OrderCount > 10;

CTEs exist only for this query and are automatically cleaned up, unlike temporary tables.
