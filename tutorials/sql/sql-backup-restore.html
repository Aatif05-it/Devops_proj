SQL Backup & Restore

Backups create database copies for disaster recovery, while restore operations recover data from those backups to a specific point in time.

Full, differential, and transaction log backups provide different recovery strategies balancing storage and recovery time.

MySQL backup and restore

mysqldump creates logical backups; mysql restores them. 

-- Full database backup
mysqldump -u root -p mydatabase > mydatabase_backup.sql

-- Single table backup
mysqldump -u root -p mydatabase Customers > customers_backup.sql

-- Restore backup
mysql -u root -p mydatabase < mydatabase_backup.sql

Logical backups are portable across MySQL versions but slower for large databases.

SQL Server backup strategies

SQL Server supports full, differential, and transaction log backups with point-in-time recovery. 

-- Full backup
BACKUP DATABASE MyDatabase 
TO DISK = 'C:\Backups\MyDatabase_Full.bak';

-- Differential backup (changes since last full)
BACKUP DATABASE MyDatabase 
TO DISK = 'C:\Backups\MyDatabase_Diff.bak'
WITH DIFFERENTIAL;

-- Transaction log backup
BACKUP LOG MyDatabase 
TO DISK = 'C:\Backups\MyDatabase_Log.trn';

Full + differential + log chain enables recovery to any transaction within the backup window.

Restoring SQL Server backups

Restore sequence: full → differentials → logs, with NORECOVERY between steps. 

-- Restore full backup
RESTORE DATABASE MyDatabase 
FROM DISK = 'MyDatabase_Full.bak'
WITH REPLACE, NORECOVERY;

-- Apply differential
RESTORE DATABASE MyDatabase 
FROM DISK = 'MyDatabase_Diff.bak'
WITH NORECOVERY;

-- Apply transaction log (point-in-time)
RESTORE LOG MyDatabase 
FROM DISK = 'MyDatabase_Log.trn'
WITH RECOVERY, STOPAT = '2025-12-22 10:30:00';

WITH RECOVERY brings the database online; STOPAT recovers to exact time.

Automated backup script

Schedule regular backups using cron (Linux) or Task Scheduler (Windows). 

#!/bin/bash
# Daily backup script
DATE=$(date +%Y%m%d)
mysqldump -u root -p$DB_PASS mydatabase > /backups/mydatabase_$DATE.sql
gzip /backups/mydatabase_$DATE.sql

# Keep 7 days
find /backups -name "mydatabase_*.sql.gz" -mtime +7 -delete

Automation with retention policies ensures consistent protection without manual intervention.
