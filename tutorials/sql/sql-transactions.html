SQL Transactions

A transaction is a group of SQL statements treated as a single unit of work, where either all changes are committed or all are rolled back.

Transactions protect data consistency for multiâ€‘step operations such as money transfers or inserting related rows in several tables.

BEGIN, COMMIT, ROLLBACK

Most databases use START TRANSACTION or BEGIN, followed by COMMIT to make changes permanent or ROLLBACK to undo them. 

START TRANSACTION;

UPDATE Accounts
SET Balance = Balance - 1000
WHERE AccountID = 1;

UPDATE Accounts
SET Balance = Balance + 1000
WHERE AccountID = 2;

COMMIT;  -- or ROLLBACK; on error

If any step fails, rolling back restores the database to its state before the transaction, preventing partial updates.

Errorâ€‘handling pattern

Application code typically checks for errors and decides whether to commit or roll back the current transaction. 

START TRANSACTION;

-- do work here

IF error_detected THEN
  ROLLBACK;
ELSE
  COMMIT;
END IF;

Explicit transactions and careful error handling ensure that related changes succeed or fail together, preserving integrity.
