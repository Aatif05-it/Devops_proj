SQL Tables (CREATE)

Tables store data as rows and columns, and are created with the CREATE TABLE statement by listing each column and its data type.

A good table design includes a primary key column that uniquely identifies every row and uses appropriate data types for each attribute.

Create a simple table

This example creates a Customers table with a primary key and some basic columns. 

CREATE TABLE Customers (
  CustomerID   INT PRIMARY KEY,
  CustomerName VARCHAR(100) NOT NULL,
  City         VARCHAR(50),
  Country      VARCHAR(50)
);

The PRIMARY KEY constraint on CustomerID ensures each customer has a unique identifier, while NOT NULL prevents missing names.

Table with auto-increment key

Many databases support an auto-incrementing integer key to generate ids automatically. Syntax varies slightly across systems. 

-- MySQL-style auto-increment
CREATE TABLE Orders (
  OrderID   INT AUTO_INCREMENT PRIMARY KEY,
  OrderDate DATE NOT NULL,
  CustomerID INT NOT NULL
);

Auto-increment keys reduce the chance of duplicate identifiers and simplify inserts because you do not need to specify the key manually.
