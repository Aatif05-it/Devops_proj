SQL INNER JOIN

INNER JOIN returns rows where the join condition matches in both tables and discards rows without a match.

It is the most commonly used join, ideal for combining entities that must have a corresponding record, such as orders with valid customers.

Orders with customer names

This query lists orders along with the name of the customer who placed each one. 

SELECT
  o.OrderID,
  o.OrderDate,
  c.CustomerName
FROM Orders AS o
INNER JOIN Customers AS c
  ON o.CustomerID = c.CustomerID;

If an order has a CustomerID that does not exist in Customers, that row is omitted from the result set.

Inner join with additional filter

Joins can be combined with WHERE to further restrict which joined rows are returned. 

SELECT
  o.OrderID,
  o.OrderDate,
  c.CustomerName,
  c.Country
FROM Orders AS o
INNER JOIN Customers AS c
  ON o.CustomerID = c.CustomerID
WHERE c.Country = 'India';

Here only orders from customers in India are listed, even though the join condition may match customers from other countries.
