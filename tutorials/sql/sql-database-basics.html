SQL Database Basics

A relational database organizes data into related tables, where each table represents one entity such as customers, orders, or products.

Tables are linked using primary keys and foreign keys so that queries can join rows across tables and keep data consistent.

Simple database model

A common beginner database has tables like Customers, Orders, and Products connected by key relationships. 

-- Example tables in a simple shop database

Customers(CustomerID, CustomerName, City, Country)
Orders   (OrderID, OrderDate, CustomerID, TotalAmount)
Products (ProductID, ProductName, UnitPrice)
OrderDetails (OrderID, ProductID, Quantity, UnitPrice)

In this model the Orders table references Customers using CustomerID, and OrderDetails links each order to specific products.

Connecting to a database

When working with SQL you typically connect to a database server (like MySQL or PostgreSQL) and then choose which database to use. 

-- MySQL-style syntax
SHOW DATABASES;

USE ShopDB;

The USE statement selects the active database so that subsequent commands such as CREATE TABLE or SELECT operate on that schema.
