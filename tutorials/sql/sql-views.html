SQL Views

A view is a stored SELECT query that behaves like a virtual table, letting you reuse complex queries with a simple name.

Views are useful for security, encapsulating joins, and providing a stable interface even if underlying tables change.

Create and use a view

This example creates a view that exposes only customers from India and then queries that view. 

CREATE VIEW IndiaCustomers AS
SELECT CustomerID, CustomerName, City
FROM Customers
WHERE Country = 'India';

-- Later:
SELECT * FROM IndiaCustomers;

Because the view stores only the query definition, it always shows the latest data from the base table when you run a SELECT.

View over multiple tables

Views can join several tables to create convenient readâ€‘only reporting layers. 

CREATE VIEW OrderSummary AS
SELECT
  o.OrderID,
  o.OrderDate,
  c.CustomerName,
  o.TotalAmount
FROM Orders AS o
JOIN Customers AS c
  ON o.CustomerID = c.CustomerID;

Applications can query OrderSummary as if it were a table, without repeating the join logic in every query.
