SQL Permissions & Users

Permissions control what users can do with database objects using GRANT and REVOKE, following the principle of least privilege.

Common privileges include SELECT, INSERT, UPDATE, DELETE, and EXECUTE on tables, views, and procedures.

Creating users and granting permissions

First create a user account, then assign specific privileges on targeted objects. 

-- Create a new user
CREATE USER 'app_user'@'localhost' IDENTIFIED BY 'secure_password';

-- Grant specific permissions
GRANT SELECT, INSERT, UPDATE ON Orders TO 'app_user'@'localhost';
GRANT SELECT ON Customers TO 'app_user'@'localhost';
GRANT EXECUTE ON PROCEDURE CreateOrder TO 'app_user'@'localhost';

-- Apply changes
FLUSH PRIVILEGES;

This user can read customers and manage orders but cannot delete data or access other tables.

Role-based permissions

Roles group permissions for easier management across multiple users. 

-- Create roles
CREATE ROLE 'readonly_role', 'order_manager_role';

-- Assign permissions to roles
GRANT SELECT ON *.* TO 'readonly_role';
GRANT SELECT, INSERT, UPDATE, DELETE ON Orders.* TO 'order_manager_role';

-- Assign roles to users
GRANT 'readonly_role' TO 'reporter'@'localhost';
GRANT 'order_manager_role' TO 'app_user'@'localhost';

FLUSH PRIVILEGES;

Users inherit all permissions from their assigned roles, simplifying administration.

Revoking and checking permissions

Use REVOKE to remove access and SHOW GRANTS to audit current permissions. 

-- Revoke specific permission
REVOKE DELETE ON Orders FROM 'app_user'@'localhost';

-- Check what permissions a user has
SHOW GRANTS FOR 'app_user'@'localhost';

-- Revoke entire role
REVOKE 'order_manager_role' FROM 'app_user'@'localhost';

Regular audits with SHOW GRANTS ensure users only have necessary access.

Column-level permissions

Grant access to specific columns instead of entire tables for extra security. 

GRANT SELECT (CustomerID, CustomerName) ON Customers 
TO 'limited_user'@'localhost';

GRANT UPDATE (Email, Phone) ON Customers 
TO 'support_user'@'localhost';

Column-level grants protect sensitive data like salaries or personal info while allowing necessary operations.
