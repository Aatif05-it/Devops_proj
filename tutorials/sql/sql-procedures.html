SQL Stored Procedures

Stored procedures are precompiled SQL routines stored in the database, callable by name with parameters for reusable business logic.

They improve security, performance, and maintainability by centralizing complex operations on the server.

Creating a procedure

This procedure creates an order and updates inventory in a single transaction. 

CREATE PROCEDURE CreateOrder(
  IN customer_id INT,
  IN product_id INT,
  IN quantity INT
)
BEGIN
  START TRANSACTION;
  
  INSERT INTO Orders (CustomerID, ProductID, Quantity)
  VALUES (customer_id, product_id, quantity);
  
  UPDATE Products 
  SET Stock = Stock - quantity 
  WHERE ProductID = product_id;
  
  COMMIT;
END;

The BEGIN...END block contains the procedure logic, with IN parameters passed when calling.

Calling procedures

Execute with CALL (MySQL) or EXEC (SQL Server), passing required arguments. 

CALL CreateOrder(101, 25, 3);
-- Creates order #auto-generated, deducts 3 from Product 25 stock

Applications call procedures instead of embedding multi-step SQL, reducing network traffic and errors.
