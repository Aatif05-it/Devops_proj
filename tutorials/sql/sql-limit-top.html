SQL LIMIT / TOP

Many databases provide a way to restrict how many rows a query returns using LIMIT, TOP, or FETCH FIRST.

Limiting rows is useful for pagination, dashboards, and testing queries on a small subset of data.

LIMIT in MySQL / PostgreSQL

This example returns only the first 5 rows from the Customers table. 

SELECT *
FROM Customers
ORDER BY CustomerName
LIMIT 5;

Adding ORDER BY before LIMIT ensures the â€œfirstâ€ rows are consistently defined, for example the alphabetically earliest customers.

TOP in SQL Server

SQL Server uses the TOP keyword to return a fixed number or percentage of rows. 

SELECT TOP 10 *
FROM Orders
ORDER BY OrderDate DESC;

This query returns the 10 most recent orders because the rows are sorted from newest to oldest before applying TOP.
