SQL Performance Tuning

Performance tuning optimizes slow queries using indexes, query refactoring, statistics, and execution plans.

Start with EXPLAIN to analyze query plans, then target missing indexes and inefficient patterns.

EXPLAIN query plans

EXPLAIN reveals how the database executes a query, showing index usage and row scans. 

EXPLAIN SELECT * FROM Orders 
WHERE CustomerID = 123 
  AND OrderDate >= '2025-01-01';
-- Look for: index usage, row estimates, join order

Plans showing "full table scan" or high row counts indicate optimization opportunities.

Common optimizations

Rewrite correlated subqueries as joins, limit result sets early, and avoid functions on indexed columns in WHERE. 

-- SLOW: Function on indexed column
SELECT * FROM Orders 
WHERE YEAR(OrderDate) = 2025;

-- FAST: Range on indexed column
SELECT * FROM Orders 
WHERE OrderDate >= '2025-01-01' 
  AND OrderDate < '2026-01-01';

Regular index maintenance and updated statistics keep the query optimizer making good decisions.
