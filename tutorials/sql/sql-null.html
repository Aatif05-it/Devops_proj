SQL NULL & IS NULL

In SQL, NULL represents an unknown or missing value, not zero, an empty string, or a space.

Because NULL is a special marker, comparisons use IS NULL or IS NOT NULL instead of the usual equality operators.

Finding NULL values

This example returns customers whose email has not been filled in yet. 

SELECT CustomerID, CustomerName, Email
FROM Customers
WHERE Email IS NULL;

Using = NULL would not match any rows; only IS NULL and IS NOT NULL correctly test for missing values.

Exclude NULL values

You can also filter out NULLs to work only with rows that have valid data in a given column. 

SELECT CustomerID, CustomerName, Phone
FROM Customers
WHERE Phone IS NOT NULL;

Being explicit about whether queries should include or exclude NULLs avoids subtle bugs in reports and aggregations.
