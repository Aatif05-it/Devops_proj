SQL UNION & UNION ALL

UNION and UNION ALL combine the results of multiple SELECT statements into a single result set.

UNION automatically removes duplicates, while UNION ALL keeps all rows and is usually faster because it skips deâ€‘duplication.

UNION example

This query returns a distinct list of cities where either customers or suppliers are located. 

SELECT City FROM Customers
UNION
SELECT City FROM Suppliers
ORDER BY City;

Because UNION applies a distinct step, each city appears only once even if it exists in both tables.

UNION ALL example

UNION ALL simply stacks results, keeping duplicates, which is useful when you will aggregate later. 

SELECT City FROM Customers
UNION ALL
SELECT City FROM Suppliers;

Because no distinct operation is performed, UNION ALL can significantly improve performance on large datasets.
