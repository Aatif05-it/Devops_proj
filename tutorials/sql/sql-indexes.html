SQL Indexes

Indexes are data structures that speed up searches and sorts on columns, at the cost of extra storage and slightly slower writes.

They are especially useful on columns frequently used in WHERE, JOIN, and ORDER BY clauses.

Create a simple index

This example adds an index on the LastName column of the Persons table. 

CREATE INDEX idx_lastname
ON Persons(LastName);

With this index, queries that filter or sort by LastName can locate rows much faster than scanning the whole table.

Composite index

A composite index covers multiple columns and is useful when queries often filter on the same combination of fields. 

CREATE INDEX idx_country_city
ON Customers(Country, City);

The column order matters: this index can efficiently support filters on Country alone or on Country plus City.
