SQL DEFAULT & NOT NULL

NOT NULL prevents a column from storing NULL, while DEFAULT supplies an automatic value when none is provided.

Combining them ensures important fields are never left blank and always have a useful initial value.

NOT NULL examples

This table requires both a name and registration date for each customer. 

CREATE TABLE Customers (
  CustomerID   INT PRIMARY KEY,
  CustomerName VARCHAR(100) NOT NULL,
  RegisteredAt DATE NOT NULL
);

Inserts that omit these columns or explicitly set them to NULL will fail with a constraint error.

Using DEFAULT values

Defaults help standardize new rows without requiring every client to provide all columns explicitly. 

CREATE TABLE Orders (
  OrderID     INT PRIMARY KEY,
  OrderDate   DATE DEFAULT CURRENT_DATE,
  Status      VARCHAR(20) DEFAULT 'Pending',
  TotalAmount DECIMAL(10, 2) DEFAULT 0
);
