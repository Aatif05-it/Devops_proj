SQL ACID & Isolation

ACID properties guarantee reliable transactions: Atomicity ensures all-or-nothing execution, Consistency maintains data rules, Isolation prevents interference, and Durability persists changes.

Isolation levels control how transactions see each other's changes, balancing consistency with performance.

ACID properties explained

Atomicity rolls back partial failures, consistency enforces constraints, isolation hides concurrent work, and durability survives crashes. 

-- Bank transfer transaction (Atomicity + Consistency)
START TRANSACTION;
UPDATE Accounts SET Balance = Balance - 100 WHERE ID = 1;
UPDATE Accounts SET Balance = Balance + 100 WHERE ID = 2;
COMMIT;  -- Both succeed or both fail

If the second UPDATE fails, the entire transfer is undone, keeping balances consistent.

Isolation levels

Common levels are READ UNCOMMITTED (dirty reads), READ COMMITTED (no dirty reads), REPEATABLE READ, and SERIALIZABLE (full isolation). 

SET TRANSACTION ISOLATION LEVEL READ COMMITTED;
-- Queries see only committed data from other transactions

Higher isolation prevents anomalies but may increase locking and deadlocks in busy systems.
