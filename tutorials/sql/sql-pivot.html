SQL PIVOT & UNPIVOT

PIVOT transforms rows into columns (e.g., monthly sales by year), while UNPIVOT does the reverse for reporting flexibility.

PIVOT is ideal for crosstab reports where values become column headers, common in Excel-style summaries.

PIVOT example: Sales by month

This converts monthly sales rows into columns for each year. 

-- Source data (long format):
-- Year | Month | Sales
-- 2024 | Jan   | 1000
-- 2024 | Feb   | 1200

SELECT *
FROM (
  SELECT Year, MonthName, Sales
  FROM MonthlySales
) AS SourceTable
PIVOT (
  SUM(Sales) FOR MonthName IN ([Jan], [Feb], [Mar], [Apr])
) AS PivotTable;

Result shows columns: Year, Jan, Feb, Mar, Apr with sales values pivoted across.

Dynamic PIVOT (SQL Server)

Generate column list dynamically when months are unknown at query time. 

DECLARE @columns NVARCHAR(MAX), @sql NVARCHAR(MAX);

SELECT @columns = STUFF((
  SELECT ',' + QUOTENAME(MonthName)
  FROM (SELECT DISTINCT MonthName FROM MonthlySales) m
  FOR XML PATH(''), TYPE).value('.', 'NVARCHAR(MAX)'), 1, 1, '');

SET @sql = '
SELECT Year, ' + @columns + '
FROM (
  SELECT Year, MonthName, Sales FROM MonthlySales
) src
PIVOT (SUM(Sales) FOR MonthName IN (' + @columns + ')) pvt';

EXEC sp_executesql @sql;

Dynamic SQL builds the PIVOT clause from actual data, perfect for variable categories.

UNPIVOT: Columns to rows

Convert pivoted data back to long format for analytics or inserts. 

SELECT Year, MonthName, Sales
FROM (
  SELECT Year, [Jan], [Feb], [Mar]
  FROM PivotedSales
) AS SourceTable
UNPIVOT (
  Sales FOR MonthName IN ([Jan], [Feb], [Mar])
) AS UnpivotTable;

UNPIVOT normalizes wide tables back to key-value pairs for easier processing.

Conditional aggregation (standard alternative)

Use CASE when PIVOT isn't available (MySQL, PostgreSQL). 

SELECT 
  Year,
  SUM(CASE WHEN MonthName = 'Jan' THEN Sales ELSE 0 END) AS Jan,
  SUM(CASE WHEN MonthName = 'Feb' THEN Sales ELSE 0 END) AS Feb
FROM MonthlySales
GROUP BY Year;

Manual pivoting works everywhere and gives full control over aggregation logic.
