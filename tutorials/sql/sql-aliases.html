SQL Aliases

An alias is a temporary name for a column or table used only for the duration of a query, created with the AS keyword or a space. [web:401][web:411]

Aliases make long names shorter, improve readability, and help distinguish columns when multiple tables share the same column name.

Column aliases

This example renames columns in the result set to more userâ€‘friendly labels. 

SELECT
  CustomerID   AS ID,
  CustomerName AS Customer
FROM Customers;

If an alias contains spaces or special characters, it must be quoted, for example AS "Contact Person".

Table aliases

Table aliases are especially useful in joins, where they shorten long table names and clarify which table each column comes from. 

SELECT
  c.CustomerName,
  o.OrderDate
FROM Customers AS c
JOIN Orders AS o
  ON c.CustomerID = o.CustomerID;

After defining Customers AS c, the alias c can be used anywhere in the query instead of the full table name.
