HTML Description Lists

Description lists are used for pairs of terms and descriptions, such as glossaries, FAQs, or key/value information. They use <dl>, <dt>, and <dd>.

Basic description list

Wrap the whole list in <dl>. Each term goes in a <dt> element and its description goes in a <dd> element. 

<dl>
  <dt>HTML</dt>
  <dd>The standard markup language for web pages.</dd>

  <dt>CSS</dt>
  <dd>A style sheet language used to describe the look of HTML.</dd>

  <dt>JavaScript</dt>
  <dd>A programming language that adds interactivity to web pages.</dd>
</dl>

Browsers usually display the description indented under the term, making the relationship between them clear.

Multiple descriptions per term

A term can have more than one description by following a single <dt> with multiple <dd> elements. 

<dl>
  <dt>Frontend developer</dt>
  <dd>Works with HTML, CSS, and JavaScript.</dd>
  <dd>Focuses on the user interface and experience.</dd>
</dl>

This is useful when you want to give both a short definition and an additional note or example for the same term.

Using description lists for key/value data

Description lists can also represent key/value pairs such as user profile details or product specifications. 

<dl>
  <dt>Name</dt>
  <dd>Alex Kumar</dd>

  <dt>Role</dt>
  <dd>Frontend Developer</dd>

  <dt>Location</dt>
  <dd>Bengaluru, India</dd>
</dl>

Compared to a table, description lists are simpler when you only need a single column of labels and values.
