HTML Comments & Best Practices

HTML comments are ignored by the browser and are only visible in the source. They are useful for notes, section markers, or temporary debugging.

Basic comment syntax

A comment starts with <!-- and ends with -->. Everything between these markers is not rendered on the page. 

<!-- This is a single-line comment -->

<p>Visible content on the page.</p>

<!--
  This is a multi-line comment.
  It can span several lines.
-->

Comments can be placed almost anywhere in the HTML but cannot be nested inside each other.

Using comments to mark sections

Comments can help you and other developers quickly see where major sections of the layout begin and end. 

<!-- Header section -->
<header>...</header>

<!-- Main content -->
<main>...</main>

<!-- Footer -->
<footer>...</footer>

Clear section comments are especially helpful in long pages or templates with repeated structures.

Temporarily disabling elements

You can wrap elements in comments to hide them temporarily without deleting the code. 

<!--
<div class="promo-banner">
  <p>Limited time offer!</p>
</div>
-->

<p>This content is still visible.</p>

This is useful when testing different layouts or turning features off during development.

Comment best practices

Comments should explain why something is done, not restate the obvious. Avoid putting secrets like API keys or passwords in comments, because they are still sent to the browser. 

<!-- Good: explains why -->
<!-- Keep this wrapper for legacy CSS layout support -->

<!-- Bad: repeats HTML -->
<!-- This is a paragraph -->
<p>Welcome</p>

Keep comments short, meaningful, and updated when you change the related code so they do not become misleading.
