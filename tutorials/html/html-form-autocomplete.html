HTML Form Autocomplete

The autocomplete attribute controls whether the browser may remember and prefill values in a form or for a specific input field.

Autocomplete on the form

Set autocomplete="on" or autocomplete="off" on the <form> element to affect all its controls by default. 

<form action="/signup" method="post" autocomplete="on">
  <label for="name">Full name:</label>
  <input type="text" id="name" name="name">

  <label for="email">Email address:</label>
  <input type="email" id="email" name="email">

  <button type="submit">Sign up</button>
</form>

When autocomplete is on, the browser may suggest stored values such as previous names or email addresses as the user types.

Autocomplete on individual fields

You can override the formâ€™s setting on specific inputs by adding autocomplete directly to the control. 

<form action="/login" method="post" autocomplete="on">
  <label for="username">Username:</label>
  <input
    type="text"
    id="username"
    name="username"
    autocomplete="username">

  <label for="password">Password:</label>
  <input
    type="password"
    id="password"
    name="password"
    autocomplete="current-password">

  <button type="submit">Log in</button>
</form>

Using specific tokens like username or current-password helps browsers and password managers fill in the correct values.

Disabling autocomplete

For sensitive fields, you can request that the browser not store or suggest values by setting autocomplete="off". 

<form action="/payment" method="post" autocomplete="off">
  <label for="card-number">Card number:</label>
  <input
    type="text"
    id="card-number"
    name="card-number"
    autocomplete="off">

  <label for="cvc">CVC:</label>
  <input
    type="password"
    id="cvc"
    name="cvc"
    autocomplete="off">

  <button type="submit">Pay now</button>
</form>

Some browsers may still apply their own rules for highly sensitive fields, so disabling autocomplete is only one part of securing a form.

Improving accessibility with autocomplete

Correct autocomplete tokens can make forms faster and more accessible, especially for users relying on assistive technologies. 

<form action="/shipping" method="post" autocomplete="on">
  <label for="shipping-name">Full name</label>
  <input
    type="text"
    id="shipping-name"
    name="shipping-name"
    autocomplete="name">

  <label for="shipping-address">Street address</label>
  <input
    type="text"
    id="shipping-address"
    name="shipping-address"
    autocomplete="shipping street-address">

  <label for="postal-code">Postal code</label>
  <input
    type="text"
    id="postal-code"
    name="postal-code"
    autocomplete="shipping postal-code">

  <button type="submit">Place order</button>
</form>

These tokens tell the browser what each field represents so it can offer accurate suggestions from the userâ€™s saved addresses.
