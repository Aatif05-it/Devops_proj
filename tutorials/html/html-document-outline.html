HTML Document Outline

A document outline is the logical structure of a page created by its headings and sections, which helps both users and search engines understand the content.

Headings define the outline

The <h1> to <h6> elements represent decreasing levels of section headings, where <h1> is the most important. 

<h1>Learning Web Development</h1>

<h2>HTML Basics</h2>
<h3>Text elements</h3>
<h3>Links and images</h3>

<h2>CSS Basics</h2>
<h3>Selectors</h3>
<h3>Layout</h3>

Screen readers and other tools can use this hierarchy to provide a navigable outline of the page for users.

Use one main heading

Most guidelines recommend using a single <h1> that describes the main topic of the page and then structuring sections with <h2> and below. 

<h1>JavaScript Tutorial</h1>

<h2>Getting started</h2>
<h3>What you need</h3>

<h2>Core concepts</h2>
<h3>Variables</h3>
<h3>Functions</h3>

Keeping one clear top-level heading reduces confusion for assistive technologies and keeps the outline simple.

Avoid skipping heading levels

Headings should follow a logical order, like an outline in a textbook: do not jump from <h1> straight to <h4> without intermediate levels. 

<!-- Good -->
<h1>Page title</h1>
<h2>Main section</h2>
<h3>Subsection</h3>

<!-- Avoid this -->
<h1>Page title</h1>
<h4>Random subsection</h4>

A consistent hierarchy makes it easier to scan the page and improves accessibility for keyboard and screen reader users.

Sectioning elements and outline

Elements like <header>, <main>, <article>, and <section> group related content but still rely on headings for a meaningful outline. 

<main>
  <h1>CodeNexa Academy Blog</h1>

  <article>
    <h2>Understanding Flexbox</h2>
    <p>...</p>
  </article>

  <article>
    <h2>CSS Grid in Practice</h2>
    <p>...</p>
  </article>
</main>
