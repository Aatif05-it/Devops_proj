Block vs Inline Elements

HTML elements are broadly divided into blockâ€‘level and inline elements. Understanding the difference helps you predict how elements flow on the page.

Block-level elements

Block elements start on a new line and normally stretch to fill the available horizontal space. Common examples are <div>, <p>, and heading tags (<h1>–<h6>). 

<h2>Block elements example</h2>
<p>This paragraph is a block element.</p>
<div>This div is also a block element.</div>

Each block element appears on its own line, one under another, creating the main structure of the page.

Inline elements

Inline elements do not start on a new line. They only take up as much width as their content and flow inside a block container. Examples: <span>, <a>, <strong>, <em>. 

<p>This <strong>word</strong> is bold,
and this <em>word</em> is emphasized in the same line.</p>

Inline elements are typically used for styling or linking parts of text inside paragraphs and other block containers.

Mixing block and inline

Block elements can contain inline elements, but inline elements should not usually contain block elements (it can lead to invalid HTML). 

<div>
  <p>You can have <strong>inline</strong> text inside a paragraph.</p>
</div>

<!-- Avoid this (block inside inline) -->
<span>
  <h2>This is not valid HTML</h2>
</span>

Use block elements to define sections and inline elements to format or highlight parts of the content inside those sections.
