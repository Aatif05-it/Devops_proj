HTML Form Advanced

Advanced form features include modern input types and builtâ€‘in validation attributes that help you get clean data with almost no JavaScript.

HTML5 input types

HTML5 added many input types such as email, url, number, date, and range that provide better UI and automatic validation in modern browsers. 

<form>
  <label>Email:
    <input type="email" name="userEmail" required>
  </label>
  <br>
  <label>Age:
    <input type="number" name="age" min="18" max="99">
  </label>
  <br>
  <label>Favorite color:
    <input type="color" name="color" value="#04aa6d">
  </label>
  <br>
  <label>Satisfaction:
    <input type="range" name="rating" min="1" max="10" value="5">
  </label>
  <br>
  <button type="submit">Submit</button>
</form>

The browser can show special keyboards on mobile and validate basic formats (for example a valid email address) automatically.

Required and length validation

The required, minlength, and maxlength attributes enforce that a field is filled and within a length range before the form can be submitted. 

<form>
  <label>Username:
    <input type="text" name="username"
           required minlength="3" maxlength="15">
  </label>
  <br>
  <label>Password:
    <input type="password" name="password"
           required minlength="8">
  </label>
  <br>
  <button type="submit">Create account</button>
</form>

If the user leaves a required field empty or the text is too short, the browser shows a builtâ€‘in error message and blocks submission.

Pattern attribute for custom formats

Use the pattern attribute with a regular expression when you need a specific format, such as a phone or custom ID. 

<form>
  <label>Phone (123-456-7890):
    <input type="tel" name="phone"
           pattern="[0-9]{3}-[0-9]{3}-[0-9]{4}"
           placeholder="123-456-7890" required>
  </label>
  <br>
  <button type="submit">Send</button>
</form>

When the value does not match the pattern, the browser marks the field as invalid and prevents the form from being submitted.

Fieldset and legend for grouped inputs

Group related controls with <fieldset> and label the group using <legend> to improve structure and accessibility. 

<form>
  <fieldset>
    <legend>Contact details</legend>

    <label>Name:
      <input type="text" name="name" required>
    </label>
    <br>
    <label>Email:
      <input type="email" name="email" required>
    </label>
  </fieldset>

  <button type="submit">Submit</button>
</form>

Fieldsets visually group inputs and give screen readers a clear description of what the grouped fields are about.
