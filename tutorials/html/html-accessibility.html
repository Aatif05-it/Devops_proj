HTML Accessibility Basics

Accessible HTML helps people using screen readers or keyboards use your website. Small changes like proper labels and landmarks make a big difference.

Labels for form controls

Every input should have a visible label connected with the for attribute and a matching id on the input. 

<form>
  <label for="name">Name:</label>
  <input id="name" type="text" name="name" required>
  <br>
  <label for="email">Email:</label>
  <input id="email" type="email" name="email" required>
</form>

When focus moves to an input, screen readers announce the label text, so the user knows what information to enter.[web:221][web:236]

Landmark elements for layout

Semantic elements such as <header>, <nav>, <main>, and <footer> act as landmarks, letting assistive technologies jump between key regions quickly. 

<header>
  <nav aria-label="Main navigation">...</nav>
</header>

<main>
  <h1>Page title</h1>
  <p>Main content goes here.</p>
</main>

<footer>
  <p>© 2025 CodeNexa Academy</p>
</footer>
